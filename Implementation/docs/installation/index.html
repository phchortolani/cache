Introduction

The basic requirements to use the Alphalinc Software Development Environment are a good machine with plenty of hard disk space left and a processor capable of running Cache. Typically, the hard disk space available must be enough to fit Caché (including the server component), all the tools and all the namespaces needed. The faster the developer workstation, the lower the time spent waiting for files to be loaded and compiled.

After installing Bazaar, it will be possible to host one or more shared repositories for source control. Since every developer will install Bazaar as part of the Alphalinc Software Development Environment, anyone can host shared repositories when appropriate.

This document is a guide for downloading, installing and configuring everything that is necessary to have a complete Alphalinc Software Development Environment. If you have not done so before, please take a look at the overview, to learn how all parts of ADSE fit together.

Software and tools installation

These are the steps required to setup the environment. Please pay attention to the installation order because some components require others to be already installed. When available, the install link provides detailed step-by-step instructions for each item.

  1. Install Operating System - The system has been developed and tested on Windows and on Unix CentOS (~ Red Hat), but not fully tested.
     
  2. Install Bazaar - Bazaar is the distributed version control system that the Alphalinc Development Environment uses. Open the following link for installation details: installation procedure. Please, take a quick look at the Bazaar Cheat Sheet and the Bazaar Workflows.
     
  3. Install Web Server - If your O.S. is Windows, please, install Microsoft's Internet Information Services (IIS): installation procedure. For Unix / Linux, install Apache (download it from httpd.apache.org).
     
  4. Install Intersystems Caché v 5.0.21 / v 2008.2.3 / v 2008.2.6 or Ensemble 2008.2.0DS - Perform a standard installation with 8-bit language (or Unicode for Ensemble). Copy the version you need from \\loira\Softwares\InterSystems to your computer. After copying, just double click on the file and follow the instructions, answering No when it asks for license information. After installing it, you can copy the license needed from one of our servers (Bonita, Morena, etc.) running a similar version that you installed. The license is usually located in the directory [CACHE_ENSEMBLE_INSTALL]\Mgr\cache.key. Copy this file to your Caché / Ensemble installation directory (at the same location) and restart it. Take note of the complete path of installation.
     
  5. Install 7-Zip - 7-zip is a free graphical file archiver with greater compression ratio than ZIP or RAR. The program can also handle ZIP and RAR files. It is used by the backup.cmd script that the environment creates. Copy 7-zip from \\loira\Softwares\Utilitários\Compactadores\7z465.exe (or download it from www.7-zip.org) to your computer. After copying it, double click on the file and follow the instructions.
     
  6. Install Java SE 6 Update 14 or higher - This package includes Java Virtual Machine and the Java Software development kit. They are required by the Tomcat Servlet Container, the ANT tool, the compiler and the jasper Reports libraries. It can be downloaded from java.sun.com. (the off-line installation version is preferred, because it is faster to complete the process, and if a reinstall is required the file has already been downloaded). Run these steps to install it: installation procedure.
     
  7. Install ANT v 1.7.1 or higher ant ant-contrib - The Apache ANT tool is the main scripting tool for the environment. The Ant-Contrib project is a collection of tasks to enhance Apache Ant. Run these steps to install both at the same time: installation procedure.
     
  8. Install Tomcat 5.5.x or Tomcat 6.0.x - The Apache Tomcat Servlet Container. Run these steps to install it: installation procedure.
     
  9. Install iReport - iReport is a graphical report designer for Jasper Reports. It can be copied from \\loira\Softwares\Desenvolvimento\Java\iReport (or download it from http://jasperforge.org). This will be a Windows executable file (iReport-2.0.3-windows-installer.exe or iReport-nb-3.5.2-windows-installer.exe). Remember that projects using Caché 5 (e.g. SES-DF) should use an iReport 2 version, and projects using Caché 2008 or Ensemble (e.g. TCI and SESAU) should use an iReport 3 version. Double click on it and follow the instructions. A good overview of how iReport and Jasper Reports work can be found at http://ireport.sourceforge.net/cap3.html. The Jasper Report libraries are already in the Alphalinc source control repository, so you must not download them separately.
     
  10. Install Winmerge - Winmerge is a free graphical diff and merging tool for Windows. Run these steps to install it: installation procedure.
     
  11. Finally, follow the instructions for project and namespace configuration for every namespace you want to work with in your workstation.

Project and namespace configuration

A developer may work with as many namespaces as needed in his or her workstation . Each namespace is associated to a different project directory. All files from a particular namespace are stored in one particular directory. To make things more organized, it is suggested a standard deployments directory be created so that each deployed project is contained there. A deployed project is one that has all its relevant files deployed to the correct places (images, CSPs, htmls, MAC, CLS, etc).

It may also be productive to work with more than one namespace for the same variation of the codebase. For example, a developer may have one project (and namespace) set to the DEV branch and another project (and associated namespace) set to the PRODUCTION branch.

Please follow the steps bellow for each namespace that you want to work with. The only limit is the available RAM and disk space on the workstation. It is very important, however, to remember to use a different name for each new namespace (remember that the same name used for the namespace is also used for the project directory, deployment directory, virtual directories and URLs).

  1. Create deployment directories - these directories will hold all the deployed files that are kept outside Caché database when Alphalinc runs. For the sake of organization, it is recommended to create a base deployment directory for every project and keep all other subdirectories inside that. However, the deployment directories can be anywhere as long as there is one deployment directory (Resources, Csps, etc) for each project. Run these steps to configure it: configuration procedure.
  2. Configure the web server (IIS) - two virtual directories will be added to the current installed web server: one for CSP files and other for regular files (HTML, images, etc). This guide will use MS Internet Information Services (IIS) as an example. Run these steps to configure it: configuration procedure.
  3. Configure a new namespace in Caché - Caché needs a few definitions to find the resource, database and CSP files. Run these steps to configure it: configuration procedure.
  4. Configure project - in this step, a new project directory is created to hold all files related to a namespace (but only to this namespace). From this directory, all files will be synchronized with one or more repositories. They will be also imported into Caché, deployed to the correct deployment directories that correspond to the locations of resources, CSPs, HTMLs, JS, reports, etc. After it is created, the directory will be bound to a bazaar repository so that contents can be synchronized the other way round. Run these steps to configure it: configuration procedure.
  5. Edit the user-config.properties file. Open the projectDir/user-config.properties file with Wordpad and fill in the properties. If you don't see the file, please review the last steps of the project configuration document.
  6. Load the source control hooks into Studio - this step differs depending on what will be done with this system. If you will use an already exported namespace (a namespace that was migrated to source control) you can just run ANT one more time and it will load all files into Caché. However, if you are migrating an existing Caché namespace to source control, don't run ant again because in this case you don't want to overwrite the existing namespace. For either case, see the document ASDE use.

Using the tools

Now that everything has been installed, you can start using the system.

Notice that some scenarios can cause the system to redeploy all files. This is a lengthy process. It can happen whenever one of the following situations occur:

  1. ANT is run for the first time after installation (because there is no "build/last-updated.log file"), just after updating all files from the first time;
  2. When the file build/last-updated.log is deleted and the global ^SourceControlTimeStamps is removed.

To see examples of tool usage, please see ASDE use.
