Alphalinc Software Development Environment (ASDE) - ASDE vs Core vs Customization

ASDE saves each type of information using specific file extensions:

  • @net Manager - classes - \globals\nm\classes\*.nmclass
  • @net Manager - classes - customization part - \globals\nm\classes\*.nmcustomclass
  • @net Manager - forms - \globals\nm\forms\*.form
  • @net Manager - forms - customization part - \globals\nm\classes\*.customform
  • @net Manager - language texts - \globals\nm\language\*.languagetext
  • @net Manager - menus - \globals\nm\menus\*.menu
  • @net Manager - favorites - \globals\nm\favorites\*.favorites
  • @net Manager - system parameters - \globals\nm\sysParameters\*.parameters
  • @net Manager - application parameters - \globals\nm\appParameters\*.parameters
  • @net Manager - user parameters - \globals\nm\userParameters\*.parameters
  • @net Manager - char parameters - \globals\nm\charParameters\*.parameters
  • @net Manager - dictionary - \globals\nm\dictionary\*.parameters
  • @net Manager - others - \globals\nm\general\*.metadata
  • Source code - includes - \src\inc\*.inc.xml
  • Source code - routines - \src\mac\*.mac.xml
  • Source code - classes - \src\cls\*.cls.xml
  • Source code - projects - \src\prj\*.prj.xml

While configuring the user-config.properties included with ASDE, there are some settings that affect which information ASDE will be responsible to handle:

  • buildOptionVAR - 1 = All files, 2 = Core only, 3 = Customization only
  • buildOptionASDE - 1 = ASDE included, 0 = No ASDE files

The files ASDE can handle are divided in three different groups: ASDE source code, Core files and customization files. The following list identifies the files that are associated to each group:

  • ASDE:
    - Source code classes from package 'SourceControl'

  • Core:
    - Source code includes (INC) and routines (MAC) not starting with 'VAR'
    - Source code classes (CLS) from package different than 'User' and 'Report' not starting with 'VAR' or from package 'User' or 'Report' not starting with 'VAR'
    - @net Manager metadata, excluding classes customization (NMCUSTOMCLASS) and forms customization (CUSTOMFORM), with the ID not starting with 'VAR'

  • Customization:
    - Source code includes (INC) and routines (MAC) starting with 'VAR'
    - Source code classes (CLS) from package different than 'User' and 'Report' starting with 'VAR' or from package 'User' or 'Report' starting with 'VAR'
    - @net Manager metadata with the ID starting with 'VAR' and all classes customization (NMCUSTOMCLASS) and forms customization (CUSTOMFORM)

* Metadata ID can be the class or the form name, the menu name, etc.

Automatic generated source code are exceptions to these rules because they are ignored by ASDE:

  • Projects (PRJ) starting with 'Default' or 'user-'
  • System includes (INC) and classes (CLS) starting with '%'
  • Every INT routine
  • Routines (MAC) starting with 'CacheSql' or 'idx.'
  • Routines (MAC) generated by persistent classes (*.T9 or *.S9)
  • Routines (MAC) generated by code method generators (*.G9)
  • Classes (CLS) generated by Web Service Servers (%SOAP.WebService) or Clients (%SOAP.WebClient)
  • Classes (CLS) starting with 'csp.'
  • Classes generated by CSP's that don't start with 'csp.'
