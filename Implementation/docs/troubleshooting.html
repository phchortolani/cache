Introduction

This is a list of the most common problems that may occur. If you can't find your problem in the following list, it may be a new one, so it should be added to the appropriate place. Search this list with Ctrl+F and the error message you have read. Alternatively, browse by topic.

General problems

There is a global that can be used to prevent the system from exporting @NM artifacts. This is helpful on production servers, where it is not interesting to run all tests that are fired from the source control hooks. If the global ^SourceControl("ignoreExport")=1 is set, the system skips @NM checks for exports.

Sometimes, during system crashes or interruptions, the ant system might be left in an incomplete state. You may force ant to redeploy part or all of its files by deleting the appropriate files from the build folder. Ant will send a list of files to be imported to Caché. Ant's list is based on the timestamps of files. Any file that was modified after the log files are put in that list. Caché gets that list and compares it to the ^SourceControlTimeStamps(filename) global. The file is actually imported only  when the timestamp is newer than that in the global. To force the tools to reimport everything, you will need to erase the build folder and also kill the ^SourceControlTimeStamps global.

File Purpose If you delete it
cache-config-script_[win|nix].s This is the Caché Terminal Script used to configure the Source Control parameters and the basic @NM parameters for correct operation in Caché. If this file is missing, ANT will regenerate it with the current values for the local folder and user-config.properties.
cache-config-script.log This file contains the log of the initial configuration made by ANT to Caché. If this file is missing, ANT will run the build\cache-config-script.s again.
cache-exclude-list_[win|nix].s This is the Caché Terminal Script that is used to configure the exclusion list global ^SourceControl("excludeList"). If the file is missing, ANT will regenerate it based on the cache-exclude-list.template.
cache-exclude-list.log The configuration of the artifact exclusion list is logged in this file. If this file is missing, ANT will regenerate the global ^SourceControl("excludeList").
update-source-control_[win|nix].s This is the Caché Terminal Script that is used to import the Source Control Package into Caché. It is a different process from the cache-update, because cache-update uses those files. If the file is missing, ANT will regenerate it based on your local configurations.
update-source-control.log This is the log of the latest Source Control Package update. It is here only for debugging purposes. Removal of this file has no implications. It is only for documentation purposes.
source-control-last-update.log This file is just a flag to hold the latest Source Control Package update timestamp. Any source control file with a timestamp later than this triggers all source control files to be imported into Caché by ANT. If this file is missing, ANT assumes that ALL source control files are candidates to be imported back into Caché.
cache-update.s_[win|nix] This is the Caché Terminal Script that is used to send all changed files (expect for the source control package) to Caché. If the file is missing, ANT will regenerate it based on your local configurations.
cache-update.log The whole process of importing artifacts into Caché is logged in this file. There are two phases. It is normal to have some files that are broken (i.e. don't compile). Removal of this file has no implications. It is only for documentation purposes.
last-updated.log This file is just a flag to hold the latest timestamp. Any file with a timestamp later than this one is a candidate to be imported into Caché by ANT. If this file is missing, ANT assumes that ALL files are candidates to be imported back into Caché. (see ^SourceControlTimeStamps).
Global: ^SourceControlTimeStamps This global holds the latest timestamps for each file imported into or exported from Caché. Any file sent by ANT is only imported if its timestamp is greater than the timestamp already in ^SourceControlTimeStamps. If this global is missing, Caché will import the whole list of files supplied by ANT. The global is recreated while files are exported or imported.

Tomcat problems

The main sources of information in the Tomcat log folder. Take a look at all files there to see if there are error messages. If there are no messages, try reaching the basic tomcat page by pointing the browser to http://localhost:8080.

If you see any of the following error messages, you may try the provided solution:

Message Solution
Could not connect to the database: Invalid Username/Password You must create the user Jasper reports will be using in Caché's SQL Manager. Take a look inside the JasperServlet.java file to see which user and password will be needed. Remember to also grant the user "jasper" the correct rights. In Caché's SQL manager, execute the following query: GRANT SELECT ON * TO JASPER.
Error Error loading byte data : http://localhost/Alphalinc/LogoSES.jpg The reports have been made by hardwiring the reference to this image inside them. While this issue does not get fixed, it is necessary to create a virtual directory in ISS host with the name Alphalinc and place this image there. Another option is to create a virtual directory in ISS host with the name Alphalinc, and then point it to the resources folder (the required file is already there). Update: we are changing the reports, one by one. Instead of loading the image from http://localhost/Alphalinc/LogoSES.jpg, they will be loaded from the URL $P{folder}+"LogoSES.jpg" (you just have to add the parameter folder to the list of parameter definitions of the report).
[SQLCODE: <-372>:<Support for extrinsic function calls are disabled>] You must enable Support for extrinsic function calls (Cache routines called within SQL statements) in the Caché configuration panel.
