Wie können Übersetzungen am effektivsten vorgenommen und mehrsprachige Anwendungen programmiert werden?

 

 Beim Anlegen von Objekten, Formularfeldern, Formulareinzelfeldern, Hilfetexten, Listen, Überschriften etc. befindet sich in @net Manager rechts oben ein Button mit dem Symbol für "Sprachen". Klickt man diesen Button an, führt @net in eine Erfassungsmaske, in der zunächst die gewünschte Sprache ausgewählt werden und dann der Begriff in der jeweiligen Sprache erfaßt werden kann. So erlaubt @net Manager, daß direkt beim Anlegen von Feldern etc. die Übersetzung dazu mit erfaßt wird. Für Übersetzungen in manuellen Routinen ermöglicht @net Manager zunächst im Menüpunkt "Sprachentexte" die Erfassung von Textnummern und den zugeordneten Wörtern/Texten. In den Routinen werden dann die Textnummern eingefügt. Beispiel: W $$^WWWTEXT(1000). @net Manager erlaubt darüberhinaus die nachträgliche Übersetzung der angelegten Objekte, Formularfelder, Überschriften usw.: Im Menüpunkt "Dienstprogramme", "Übersetzungen erfassen" können die gewünschten Sprachen angeklickt werden. @net Manager sucht dann alle noch nicht übersetzten Wörter und listet diese auf. Die Übersetzung kann dann in das darunterstehende Feld eingegeben werden. Sofern @net Manger den zu übersetzenden Begriff bereits im umfassenden Wörterbuch gespreichert hat, wird die neue Übersetzung vorgeschlagen und braucht lediglich bestätigt - oder geändert - zu werden.
