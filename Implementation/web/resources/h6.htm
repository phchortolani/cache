Hilfe

Wie generiert man Formulare?

@net Manager kann Formulare automatisch aus den Datensatzdefinitionen erstellen. Beispiel für ein automatisch erstelltes Formular: Menüpunkt "Lager" - Untermenü "Lagerstammdaten" - darunter "Erfassen Lagerparameter". Zur automatischen Erstellung gibt man bei "Erstellen von Formularen und Listen" die Formularart "Standardformular" ein. Die "Basisdatei für Formular" ist die Lagerdatei, die in der Datensatzdefinition hinterlegt ist. Werden dann noch die Punkte "Autom. Formular generieren" und "Vorgabe aus Mandantenvorgabe" ausgewählt, erstellt @net Manager automatisch ein vollständiges Formular. Das Fomular generiert sich erstmalig durch Aufrufen des Buttons "Testen von Formularen". Anschließend sind die Datenfelder automatisch verfügbar, die natürlich noch manuell ergänzt werden können.

Am Beispiel des Datenfeldes "Lagerbild" wird klar, wie man Feldprüfungen ändern kann. @net Manager setzt das Feld Erfassungsart auf Automatik. Soll dies überschrieben werden mit der Erfassungsart "File", wodurch automatisch der Button "Durchsuchen..." erscheint, wählt man im Feld Erfassungsart die Option "File".

Das Programm "Erfassen Lagerparameter" ist außerdem ein Beispiel dafür, was das Feld "Option" bedeutet. Wird z.B. die Option "mit Lagerbild" ausgewählt, generiert @net Manager beim Aufrufen des Formulars automatisch ein Foto des Lagers. Wenn optional "ohne Lagerbild" gewünscht, erscheint das Bild des Lagers nicht.

Wie in @net Manager JavaScript-Anwendungen eingefügt werden können, wird ebenfalls im Programm "Erfassen Lagerparameter" deutlich (siehe Feld "Anzahl Wagengößen").

Weitere Beispiele und Erläuterungen zum Erstellen von Formularen finden Sie in den entsprechenden Hilfetexten.

@net Manager ermöglicht, in automatisch erstellten Formularen Individualroutinen aufzurufen. Im Menüpunkt "Erfassen Formulare/Listen" kann in den Feldern "Execute nach Speichen Datensatz" und "Execute nach Löschen Datensatz" die Individualroutine hinterlegt werden, die beim Speichern bzw. Löschen gestartet werden soll (siehe Hilfetexte). In der Demo-Version finden Sie das Beispiel "Ein-, Aus- und Umlagern von Wagen". Basisdatei für Formular ist hier die Datei "QLXZ (Lagerbewegungsdatei)". Aufgerufen wird die Individualroutine "QLXZ" zur Prüfung von freien Lagerplätzen etc.

Formulare mit Buttons

Das Formular "Ein-, Aus- und Umlagern von Wagen" ruft durch Klicken auf den Button "Lageranweisungen" ein anderes Formular (Zielformular) auf. Damit das funktioniert, muß zunächst der Button "Lageranweisungen" erstellt werden. Er besteht aus einer Bilddatei (GIF-File), die in einem beliebigem Grafikprogramm erstellt werden kann. In "Erstellen von Formularen" wird der Button "Button" angeklickt, um die Datei des neu erstellten Buttons "Lageranweisungen" im Feld "Bilddatei für Button" und die Bezeichnung des Zielformulars einzutragen.

Formulare ohne Buttons

Soll ein Formular ohne Button erstellt werden, ist keine Basisdatei für das Formular einzutragen (Bsp.: Formular "Lageranweisungen ONLINE"). Man kann entweder freie Felder eintragen (Button "Freie Felder" anklicken) oder direkt eine Individualroutine (M-Routine im Feld "Execute bei Formularstart") aufrufen.

Manuelle Formulare

Wird ein manuelles Formular gewünscht, muß als Formular-Art "Freie Felder (mit Button)" ausgewählt werden. Im Feld "Execute nach Speichern Datensatz" wird die Individualroutine eingetragen, die nach dem Auswählen des Buttons "o.k." aufgerufen werden soll. Dann werden die freien Felder angelegt. Das erfassen der manuellen Felder geschieht im gleichen Programm wie das anlegen und Ändern von Datenfeldern.

Ein Beispiel für ein manuelles Formular ist "Lagerübersicht". Nach der Auswahl des Buttons "Datenfelder" wird im "Erfassen Datenfelder" im Feld "Manueller Fragetext" eingegeben, welcher Fragetext später im Programm erscheinen soll. Im Feld "Inhalt in Variable" ist die Variable einzutragen, die @net Manager später an das Programm abgeben soll (hier z.B. "VORG(1)). Das Programm, in dem dann die Bearbeitung erfolgt, ist die Routine "QLXU". Ausserdem können Sie noch die Erfassungsart / Länge und Relationsdateien angegeben werden. Um ein einmaliges Auswahlfeld zu erhalten können Sie auch Erfassungsalternativen erfassen. (Button: Selektionen)

Unterformulare

@net Manager erlaubt die Funktionalität des Wechselns in ein Unterformular durch einfaches Anklicken eines entsprechenden Buttons. Das Formular "Lagerparameter" beispielsweise enthält einen Button "Lagerplätze", durch dessen Anklicken @net Manager in das entsprechende Unterformular wechselt. Ein weiterer Button führt durch Anklicken in das Unterformular "Lager-Wagen". Durch Betätigen des "Zurück-Buttons" führt @net Manager den Benutzer jeweils wieder in das vorherige Formular zurück.

Sollen verschiedene Formulare auf diese Art miteinander verbunden werden, ruft man zunächst das erste Formular auf (Beispiel: Formular "Lagerparameter"). In dem aufgerufenen Formular wird der Button "Button" angeklickt. @net Manager öffnet dann eine Button-Datei, in der im Feld "Button-Bezeichnung" der Name des gewünschten Buttons und im Feld "Formular bei onClick" die Datei, die aufgerufen werden soll (Beispiel "QLXP"), eingetragen wird. Anschießend ist die Bilddatei für den gewünschten Button einzutragen (Beispiel: "platz.gif"). Wird keine Bilddatei eingetragen, setzt @net Manager als Button die oben eingegebene Button-Bezeichnung ein, kreiert also einen Text-Button (z.B. der Button "Lager").

Soll beim Aufrufen eines Unterfomulars der Primärschlüssel des Vor-Formulars automatisch gesetzt werden, so ist im Feld "Fix Primärschlüssel Vor-Datei" des Unterformulars die Nummer des Primärschlüssels einzutragen. Nach dem Eintragen des Primärschlüssels ist dieser beim Aufrufen des Unterformulars fest vorgegeben.

Manuelle Programme (Formulare)

Soll in einem Formular durch Anklicken eines Buttons eine manuelle M-Routine, ein anderes Pogramm (z.B. Word, Excel etc.) oder eine andere Internet-Seite aufgerufen werden, wird diese Verbindung wie folgt vorgenommen:

In dem gewünschten Formular wird der Button "Button" angeklickt. @net Manager öffnet dann eine Button-Datei, in der im Feld "Button-Bezeichnung" der Name des gewünschten Buttons eingetragen wird. Zum Aufrufen einer M-Routine ist im Feld "Execute bei onClick" die Datei, die aufgerufen werden soll (Beispiel "D ^QLXU"), einzugeben. Soll beim Anklicken des Buttons in ein anderes Programm gewechselt werden (z.B. Excel), ist im Feld "Verknüpfung mit" der entsprechende Pfad- und Dateiname einzugeben (z.B. "C:/TEST.DOC"). Zum Aufrufen einer Internet-Suchmaschine ist als "Verknüpfung mit" z.B. "http://www.altavista.com" einzutragen.

Anschießend ist die Bilddatei für den gewünschten Button einzutragen (Beispiel: "platz.gif"). Wird keine Bilddatei eingetragen, setzt @net Manager als Button die oben eingegebene Button-Bezeichnung ein, kreiert also einen Text-Button (z.B. der Button "Lager").

Wie werden in Formularen Individualprogramme aufgerufen?

Bei jedem Formular und jeder Liste, das/die mit @net Manager erfaßt wird ("Erfassen Formulare/Listen"), können an verschiedenen Stellen M-Routinen hinterlegt werden. Es gibt die Möglichkeit eines "Execute bei Formularstart", "Execute nach Speichern Datensatz", "Execute nach Löschen Datensatz" und "Execute nach Ändern Datensatz". Einzutragen ist in diesen Feldern lediglich z.B. "Do ^PROGRAMM). @net Manager ruft dann die entsprechende M-Routine auf.

Sollen M-Routinen oder auch JavaScript-Anwendungen für ein einzelnes Feld hinterlegt werden, klickt man zunächst in das gewünschte Feld (Beispiel: Formular QLX Lagerparameter, Feld "Datenfeld Nummer"). Anschließend klickt man zum Laden des Datensatzes auf den Öffnen-Button. Jetzt kann man entweder auf den Button M-Routinen oder JavaScript klicken. Wählt man z.B. JavaScript, öffnet sich ein Unterformular, in das JavaScript-Anweisungen für das entsprechende Feld eingetragen werden können. M-Routinen können auf konventionelle Weise oder im Unterformular "Erstellen M-Routinen" erstellt werden.

Näheres dazu ist in den Hilfetexten hinterlegt.
