Benötigt man System- u. Anwenderparameter / Relations-Dateien?

Für bestimmte Darstellungsarten sind diverse Parameter zu erfassen. Einen besonderen Stellenwert erhalten die Parameter im Hinblick auf die Mehrsprachigkeit von @netv Manager. Werden z.B. die Parameter "1" für "Herr" und "2" für "Frau" eingegeben, listet @net Manger automatisch die korrekten Anreden und die entsprechenden Pflichtfelder.

Beispiel: in der "QLXW Lager-Wagendatei" wird im ersten Datenfeld mit der Bezeichnung "Wagenbreite", hinterlegt, daß diese auf den Anwendungsparameter "BREITE" zugreifen soll. Es gibt in diesem Beispiel drei Breiten:

1 = klein, 2 = mittel, 3 = groß. Dafür ist im Feld "Relation Datei" "WWW101" einzutragen. Im darunterliegenden Feld "Relation-Primärschlüssel" wird ""BREITE",SPRACHE" eingegeben. Näheres dazu finden Sie in den Hilfe-Texten.
