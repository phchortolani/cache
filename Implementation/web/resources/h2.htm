Was ist @net Manager?

@net Manager ist eine Caché Client Server Anwendung, mit der sowohl einfache als auch komplexe, datenreiche, objektorientierte, dynamische, multilinguale und multikulturelle Internet- und Intranet-Client Server Anwendungen online erzeugt werden können.

Mit @net Manager ist Applicationsentwicklung so einfach wie Surfen im Internet! Der Browser wird als Programmierer- und Anwender Front End genutzt. Das bedeutet, die Progammierer können irgendwo auf dieser Welt online Applicationen erstellen, die wiederum weltweit sofort und online lauffähig sind.

Wie macht @net Manager das?

@net Manager erzeugt mit Hilfe von Caché in Verbindung mit Weblink dynamische HTML or XML Seiten. Aus einem Event der Seite (i.e. Button: Speichern, Neu, Löschen, Öffnen, nächster Datensatz, Verzweigungen in andere Programme/Formulare/Links) wird der gesamte Programmablauf gesteuert.

Durch das Manövern durch die einzelnen Seiten werden durch die Auswahl von Methoden, das Bestimmen gewünschter Funktionen und den Eintrag von Eigenschaften letztlich die Anwendungen erzeugt. Dabei werden keine fertigen HTML Seiten compiliert, sondern erst zur Laufzeit werden alle Objekte ausgewertet und aufgrund der Benutzer- und Modulinformation (Berechtigung, Sprache, Kultur, Farben, Formen) neue dynamische Seiten erzeugt.

Wie sind @net Manager und die @net Manager Anwendungen aufgebaut?

@net Manager bedient sich des Objekt Modells. Für @net Manager ist damit jede HTML Seite ein Objekt. Dieses Objekt hat viele unterschiedliche Eigenschaften, Methoden und wiederum Objekte. Es gibt einfache Eigenschaften wie Farben, Formen, Frames oder Buttons, aber auch komplexere wie Caché Datenbank Objekte. Wenn eine Anwendung sich auf eine Datenbank bezieht oder eine neue Datenbank angelegt werden soll (diese kann natürlich später auch als CachéObjectScript ausgegeben werden) kann @net Manager automatisch die Eigenschaften auf die Anwendung vererben. Das gleiche geschieht z.B. bei Layoutvorgaben, die von Mandant à auf die Anwendung à auf das Formular à auf das Datenfeld vererbt werden können.

Internet und Intranet werden heute von vielen Firmen und privaten Anwendern genutzt. Man hat erkannt, daß die Internet-Technik für eine Online-Verbindung preiswerter ist als die meisten herkömmlichen Netze. Da es sich beim Internet um ein weltweites Netz handelt, nimmt die Bedeutung von internationalen Anwendungen immer mehr zu. So profitieren z.B. große Softwarehäuser, die in unterschiedlichen Ländern oder auf mehreren Kontinenten Filialen haben, von Anwendungen, die mehrsprachig sind und von vielen Mitarbeitern für diverse Zwecke eingesetzt werden können. Betrachtet man allein die derzeitige Entwicklung auf dem europäischen Markt, wird deutlich, welche Vorteile eine mehrsprachige und multikulturelle Online-Anwendung hat. Die Anwender der verschiedenen Staaten brauchen sich nicht auf eine einheitliche Sprache zu einigen, sondern jeder kann sein Programm in seiner Landessprache nutzen. Auch die kulturell typische Schreibweise von Währungen, dem Datum und der Uhrzeit erleichtern die Einarbeitung von Usern erheblich.

@net Manager ermöglicht eine preiswerte Verteilung von Ressourcen. So kann z.B. der Server in Kanada stehen, die Entwicklungsabteilung in den USA ansässig sein, die Datenerfassung in Indien erfolgen und die statistische Auswertung von einer Firma in Deutschland abgerufen werden - alles online, mehrsprachig und multikulturell! Benötigt die Abteilung Statistik z.B. ein neues Datenfeld, genügt ein E-Mail in die USA. Dort wird das neue Datenfeld mit Hilfe von @net Manager auf einfache Art und Weise innerhalb von wenigen Minuten generiert. Nach dem Aufrufen der HTML Seite in Indien durch die Datenerfasser ist das neue Feld in Indien aktiviert. Durch die von @net Manager automatisch beim Entwickeln zu Verfügung gestellten Hilfefunktionen wird es allen Usern leicht gemacht, die Neuerungen unverzüglich zu nutzen.

Auch die Intranet Programmierung von Systemhäusern für Dritte wird mit @net Manager vereinfacht. Neben den unterstützenden Funktionen bei der Entwicklung gibt es einfache Prozeduren zur Auslagerung von Programmständen oder Modulen auf beliebige Medien. So lassen sich komplexe Anwendungen mit Hilfe von @net Manager erstellen und einfach supporten.

Die Arbeit mit @net Manager ist zeitsparend, der Einstieg einfach. Durch eine übersichtliche Menüführung und eine detaillierte Online-Hilfe findet man sich gut zurecht. So kann @net Manager von privaten Anwendern ohne Programmierkenntnisse genauso genutzt werden wie von kleinen, mittleren oder großen internationalen Softwarehäusern mit hohen Ansprüchen.

Wer die Browser-Technik nutzt, verwendet oft auch andere Produkte, die HTML/XML (XML in Vorbereitung) verstehen, wie z.B. Microsoft Word, Excel etc. Die Inhalte einer mit @net Manager generierten Liste lassen sich einfach in anderen Programmen (z.B. Excel) weiterbearbeiten. Die @net Manager Menüpunkte können nicht nur auf @net Manager Anwendungen linken, sondern auch auf andere Anwendungen (.EXE), andere URLs oder eine mit FrontPage erstellte lokale statische HTML Seite.

Mehr und mehr Firmen bieten ihre Produkte in Internet-Shops zum Verkauf an. Hier findet sich auch ein Einsatzgebiet von @net Manager. Wenn mit FrontPage o.ä. ein Internet-Shop erstellt wurde, kann mit @net Manager die Administration dazu kreiert werden. Das bedeutet, mit @net Manager läßt sich die gesamte Auftragsverwaltung des Shops gestalten (Bestellung, Auftragsbestätigung, Lieferschein, Fakturierung, Lagerbestandsverwaltung, Lagersteuerung etc.).

Weitere Vorteile von @net Manager:

Objektorientiert: Die benötigte Funktionalität konnte nur mit einem Objekt Modell gelöst werden. Auch für die Vererbung von Layout-Eigenschaften war diese Vorgehensweise ideal. Ein weiterer Vorteil der Objektorientiertheit ist die Ausgabe in Form von CachéOjectScripts, damit Daten und Datensatzdefinitionen in bestehenden Caché Anwendungen weiterverwendet werden können.

Mehrsprachig: Das gesamte Tool ist, wie auch die erstellten Anwendungen, mehrsprachig. Übersetzungen können manuell oder automatisch erfasst werden. Ein Übersetzungs-Tool sucht in der entstandenen Anwendung nach unvollständigen Übersetzungen und schlägt aus einer Caché Datenbank eine mögliche Übersetzung vor, die geändert und gespeichert werden kann. Der Übersetzungsprozeß schließt sogar Maskenköpfe, Datenbank-Name, Menues, Hilfetexte, Statustexte, Parameter, Seiten-Reiter und Button-Bezeichnungen ein. Alle sprachabhängigen Objekte haben eine Spracheigenschaft, die aktiv wird, sobald @net Manager die Sprache des Benutzers kennt.

Multi-Kulturell: Wie bei der Sprachkennung haben Felder (wie z.B. Datum, Uhrzeit, Betragsfelder) eine Kultureigenschaft, die aktiv wird, sobald @net Manager die Kultur des Benutzers kennt. Die Zahl "Eintausend" z.B. könnte wie folgt aussehen: englisch "1,000.00"; deutsch "1.000,00"; schweizerisch "1'000,00"; portugiesisch "1,000$00" usw. Auch Daten (amerikanisch "02/04/1999" oder deutsch "02.04.1999") und Uhrzeiten (englisch "5 pm" oder deutsch "17:00") werden je nach Kultur des Anwenders dargestellt.

Multidimensional: @net Manager ermöglicht komplexe Anwendungen und Programmverteilungsmethoden zu anderen Formularen, die Multidimensional miteinander verbunden sind, aber auch zu URL´s oder .EXE Anwendungen (i.e. Word, Fax, e-mail).

Dynamisch: Alle Teile der Anwendung (Menüs, Masken, Listen, sogar das Tool selbst) werden erst bei der Laufzeit von einem Caché Programm mit den erfaßten Maskendefinitionen, Layout-Parametern, Datensatzdefinitionen, Datenfelddefinitionen, Button-Beschreibungen, Such-Parametern, der Benutzer-Sprache, der Benutzer-Zugriffs-Berechtigung, dem JavaScript/VBScript, den Object-Sammlungen (save, open, delete, new, next record, prior record...), usw zusammengestellt. Das Ergebnis ist ein dynamisches HTML Dokument mit eingebundenem JavaScript/VBScript. Die Feldprüfungen werden sowohl von JavaScript/VBScript als auch von Caché Routinen vorgenommen.

Besonders benutzerfreundlich:. Alle Anwendungen, die mit @net Manager entwickelt werden, sind anwenderfreundlich, denn alle Eingabefelder sind darauf vorbereitet, einen Hilfetext zu tragen. In jeden Hilfetext können Bilder oder Videos eingebunden werden. Der Anwender kann sich je nach Wunsch eine gesamte Bedienungsanleitung ausdrucken oder sich die einzelnen Hilfetexte durch Klicken des Hilfebuttons auf dem Bildschirm ansehen. Auch für Programmierer stellt @net Manager eine Vielzahl von Hilfetexten und Hilfsfunktionen bereit. @net Manager kreiert z.B. automatisch Formulare und Listen aus der Datensatzdefinition und schlägt Eingabearten (Checkboxen, Radio buttons, Selektionsfelder ) vor, wenn auf Relationen verwiesen wird. Außerdem erkennt @net Manager bestimmte Eingabemuster (z.B. e-mail Adressen) und kreiert automatisch einen Button, der dann die entsprechende Funktion ausführt. Bestandteil von @net Manager sind darüberhinaus eine Reihe von kompletten Anwendungen und Hilfsprogrammen (z.B. automatisches e-mail und Fax System, Download, embeded object, Gruppenterminkalender, Datenimport- und -export-Tools, Schwarzes Brett etc.).

 

Multi-Userfähig: Da viele Anwender gleichzeitig online auf die Datenbank zugreifen können, aber der Programmjob nach dem Senden der Seite beendet ist, war ein neuer Satzlockmechanismus für diese Art von Internet-Programmierung nötig, da der bestehende Lock-Mechanismus (+L und -L) nicht verwendet werden konnte. Der neue Mechanismus ist je Datei einstellbar, individuell und zeitlich begrenzt. Nur der Besitzer des Datensatzes kann den Datensatz ändern. Das bedeutet, sobald man einen Datensatz angezeigt bekommt, ist dieser gelockt. Wenn die vorgegebene Lockzeit abgelaufen ist, kann ein neuer Benutzer an den Datensatz. Derjeige, der ursprünglich gelockt hat, kann einfach wegklicken und der Datensatz wird wieder entlockt (unlocked), obwohl die Bearbeitung via Browser ja offline ist.

Mandantenfähigkeit: Ein Server verweist mit WebLink auf einen bestimmten Benutzerbereich (Namespace) und startet entsprechende Routinen. Innerhalb der @net Manager Anwendung können beliebig viele unabhängige Firmen angelegt werden. Je Anwender kann hinterlegt werden in welcher Firma er arbeitet. So erlaubt @net Manager einer unlimitierten Zahl von Firmen und Nutzern, gleichzeitig und ohne Störung innerhalb nur eines Namespace auf die Firmen Datenbanken zuzugreifen.

Sicherheit: Bei der erfolgreichen Anmeldung eines Benutzers mit Benutzername und Password wird von @net Manager eine einmalige User-Nummer vergeben, die im weiteren Verlauf der Sitzung automatisch in die dynamischen Seiten eingebunden ist und damit in Verbindung als Berechtigung gilt. Nach Beendigung der Sitzung ist diese User-Nummer nicht mehr gültig. Ein anderer User kann dann nicht einfach mit dem Button "Zurück" oder der rechten Maustaste an die Informationen herankommen.
