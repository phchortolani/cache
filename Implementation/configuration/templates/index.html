Alphalinc Development Environment - Templates

Location: docs/templates

Templates are initial configuration files that the user needs to copy to the correct location and edit to match their current system parameters.

Please note that the templates must not be changed in place! The user have to copy the file to the correct location and then change it. When a template file is copied to the correct location and then changed it becomes a user configuration file. COnfiguration files are not under version control because every user has its own environment settings.

The configuration files are:

Template name Destination Purpose
local.properties project-base-dir\local.properties
(example: c:\soeiro\AlphalincMain\local.properties) 		This file stores the paths names and other parameters needed by the core of the build scripts.
authentication.conf Linux: $HOME/.bazaar/authentication.conf

Windows (a single line): c:\Document and Setting\USERNAME\Application Data\bazaar\2.0\authentication.conf

(USERNAME is the name of the user login to Windows)

This file handles the authentication information for the bzr commands. When it is properly setup, the user don't have to type its authentication information all the time. Complete information can be obtained here.
